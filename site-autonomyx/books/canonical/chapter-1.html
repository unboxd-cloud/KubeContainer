Fabric Framework

Fabric Framework is a delivery-first framework for turning intent into evidence-backed work across platforms, agents, tools, apps, runtimes, and SDKs.

This repository publishes the canonical framework artifacts for OpenAutonomyX.

Drift Log

This log preserves mistakes as evidence.

Mistakes are not deleted.

They are marked as drift, corrected forward, and kept visible.

Why this exists

The project theory requires evidence.

If drift is erased, the system loses the proof trail.

So drift must be published. 

promise made
proof missing
gap exposed
drift recorded
correction carried forward

Current invariant

Fabric Framework = theory
PlatformContract = contract authority
Agent = delivery obligation
Box = gap filler / carrier
Evidence = proof

Drift 1: Box was promoted into theory

Drift

Box was described as if it were the theory.

Why wrong

Box does not explain the whole fabric.

Fabric Framework is the theory.

Box only fills the gap between contract and delivery.

Correction

Fabric Framework = theory
Box = gap-filling carrier

Drift 2: Promise/proof gap was claimed before proof existed

Drift

The system claimed Box could fill the promise/proof gap before an executable delivery test proved it.

Why wrong

A claim without evidence is exactly the gap the framework must expose.

Correction

No delivery claim without evidence.
No proof claim without replay.
No success claim without report.

Drift 3: Mistakes were almost cleaned instead of preserved

Drift

Cleanup was suggested in a way that could erase evidence of wrong turns.

Why wrong

Deleting drift hides accountability.

Correction

Do not delete drift.
Publish drift.
Correct forward.

Drift 4: Noun-first instruction grammar

Drift

The grammar used noun-first forms such as: 

box.create
box.verify

Why wrong

The foundation moved toward structured instruction with instruct.box as the root.

Correction

instruct.box

with structured payload: 

verb
scope
constraints
payload

Drift 5: Imported software package language

Drift

Words such as package, packed, and published were introduced too early.

Why wrong

They carry existing software distribution meanings and polluted the Box model.

Correction

BoxPack = move form of Box
pack.box = operation
open.box = operation

Avoid package language unless explicitly approved.

Drift 6: Run language inside Box

Drift

The nested unit inside Box was called MiniRun.

Why wrong

Run is runtime-centric and not self-similar.

Correction

Box
  -> MicroBox

Current correction discipline

Before adding a new noun or feature, ask: 

What is the theory?

What proves it?
