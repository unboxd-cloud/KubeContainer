Autonomyx Partners

Open Source Commercial Partner Delivery Network

Autonomyx is building a certified commercial delivery network for open source agentic infrastructure, decision intelligence, and governed automation.

This repository defines the partner model, delivery standards, certification structure, marketplace rules, governance requirements, and contribution process for the Autonomyx partner ecosystem.

Positioning

Autonomyx is not open core.

Autonomyx is an Open Source Commercial Core company:

  • The software is open source.
  • The business is commercial.
  • The delivery is partner-led.
  • The trust layer is governed.
  • Customers pay for outcomes, implementation, operations, assurance, and lifecycle ownership.

Autonomyx does not sell a weak community edition and a hidden enterprise edition. Everyone gets the real platform. Commercial value comes from trusted delivery, governance, operations, compliance, industry packaging, certification, and managed lifecycle services.

Enforcement Policy

Principle

Checks without enforcement are guidance.

Autonomyx standards become enforceable only when repository controls prevent bypassing the review process.

Required branch protection

The default branch must enable:

  • Require pull requests before merge
  • Require approval reviews
  • Require review from CODEOWNERS
  • Require status checks to pass
  • Dismiss stale approvals on new commits
  • Require conversation resolution
  • Restrict force pushes
  • Restrict branch deletion

Required workflow gates

The following checks must pass before merge:

  • Markdown validation
  • YAML validation
  • Registry validation
  • Metadata validation
  • Documentation validation
  • Certification validation
  • Conformance validation

RFC enforcement

Draft proposals must not be merged directly as standards.

Required path:

Draft RFC → Community Review → Technical Review → Governance Review → Approval → Published Standard

Certification enforcement

A partner may not claim certification unless:

  • Evidence exists
  • Review completed
  • Status published in registry
  • Certification status equals certified

Marketplace enforcement

Marketplace assets may be delisted when:

  • Security risk exists
  • Required metadata is missing
  • Publisher identity cannot be verified
  • Certification is suspended
  • Governance violations occur

Trust enforcement

Trust is earned through evidence.

Trust may be reduced or revoked when:

  • False claims are made
  • Delivery evidence is fabricated
  • Security obligations are ignored
  • Customers are abandoned
  • Governance requirements are bypassed

Merge policy

No direct commits to main.

All standards, certifications, governance changes, registry changes, marketplace policies, and review procedures must enter through pull requests.

RFC: Autonomyx Open Source Commercial Partner Delivery Network v0

Status: Draft for community review Target audience: partners, implementers, managed service providers, technology vendors, solution builders, open source contributors, enterprise buyers

Summary

This RFC proposes the first public draft of the Autonomyx Open Source Commercial Partner Delivery Network.

The goal is to define how Autonomyx can remain genuinely open source while supporting a commercial delivery ecosystem around implementation, managed services, certification, marketplace assets, governance, and customer accountability.

Core proposal

Autonomyx is not open core.

Autonomyx is an Open Source Commercial Core company:

  • The platform remains open source.
  • Commercial value comes from delivery, operations, assurance, compliance, certification, and lifecycle responsibility.
  • Partners can build businesses around implementation, integration, managed services, training, and industry solutions.
  • Customers get transparent software, governed delivery, and accountable support.

What is included in this draft

  • Partner model
  • Certification framework
  • Governance framework
  • Commercial delivery positioning
  • Partner tier structure
  • Marketplace direction
  • Initial registry structure

What needs review

We invite feedback on the following questions.

1. Partner model

  • Are the partner categories correct?
  • Are any important partner types missing?
  • Should implementation, managed services, solution, technology, training, and marketplace partners remain separate categories?

2. Certification

  • Is the certification framework practical for early partners?
  • Are the evidence requirements too heavy, too light, or appropriate?
  • Should certification be self-attested first and audited later, or reviewed before publication?

3. Governance

  • Are the governance controls strong enough to protect customers?
  • Are any controls too restrictive for open source adoption?
  • What should trigger suspension or revocation?

4. Marketplace

  • What should be allowed in the partner marketplace?
  • Should paid partner assets be listed beside free/open assets?
  • What metadata is mandatory for every listing?

5. Commercial boundary

  • Is the open source versus commercial boundary clear?
  • What must always remain open source?
  • What can be commercial without becoming open core?

6. Partner incentives

  • How should partners earn revenue?
  • Should Autonomyx charge certification fees, marketplace fees, support fees, or revenue share?
  • What incentives help small partners, regional firms, and independent experts participate?

Review instructions

Please provide feedback using GitHub issues or pull request comments.

Useful feedback formats: 

Problem:

Non-goals for this draft

This draft does not finalize pricing, legal contracts, certification fees, or partner agreements.

This draft does not create a foundation structure by itself.

This draft is the starting governance and delivery model for public review.

Acceptance criteria for v1

Before v1, the repository should include:

  • Finalized partner categories
  • Finalized partner tiers
  • Certification evidence templates
  • Marketplace listing templates
  • Delivery evidence template
  • Review process
  • Governance enforcement process
  • Contribution guide
  • Code of conduct
  • Public registry format

Current status

This is a draft. Nothing here should be treated as final policy until v1 is approved and tagged.
