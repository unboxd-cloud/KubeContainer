OpenAutonomyX Search

OpenAutonomyX Search is the retrieval and indexing layer for the Fabric platform.

It uses OpenSearch for full-text, vector, hybrid, log, and observability search while keeping Fabric / SurrealDB as the system of record.

Architecture rule

Fabric / SurrealDB = source of truth
OpenSearch        = derived search index
API               = controlled query surface
Workers           = ingestion + reindexing
Policies          = deny-by-default access filters

OpenSearch must never become the authority for identity, ownership, policy, or provenance. Every indexed document must carry Fabric metadata.

Core document fields

Every indexed record should include:

  • fabric_id
  • tenant_id
  • workspace_id
  • source_uri
  • schema_type
  • schema_version
  • title
  • content
  • tags
  • created_at
  • updated_at
  • provenance
  • acl_subjects
  • embedding

Local development

docker compose up -d
npm install
npm run setup:indexes
npm run dev

OpenSearch runs at: 

http://localhost:9200

Health check: 

curl http://localhost:9200/_cluster/health?pretty

API

The service exposes a small controlled API:

  • POST /v1/index/documents
  • POST /v1/search
  • POST /v1/reindex
  • GET /healthz

See [openapi.yaml](./openapi.yaml).

Index strategy

Default index aliases:

  • fabric-documents-read
  • fabric-documents-write
  • fabric-events-read
  • fabric-events-write

Versioned physical indexes:

  • fabric-documents-v1
  • fabric-events-v1

Security model

Search must apply tenant and ACL filters before returning results.

Minimum filters: 

{
  "tenant_id": "tenant_x",
  "acl_subjects": ["user:123", "role:admin", "agent:abc"]
}

Why OpenSearch here?

OpenSearch is used because it is Apache-2.0, open source, self-hostable, and suitable for full-text search, analytics, observability, and vector/hybrid retrieval workloads.

Status

Initial scaffold. Next step is to connect ingestion to Fabric events and enforce policy decisions before query execution.
